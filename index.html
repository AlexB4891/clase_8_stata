¡Bienvenida a la Clase 8!

En esta clase, aprenderemos a construir una base de datos para la conciliación tributaria, combinando datos de diferentes fuentes y realizando transformaciones complejas en Stata.

El do-file de hoy es un ejemplo práctico de cómo se realiza una tarea de **ETL (Extracción, Transformación y Carga)** en un entorno de análisis de datos real. Nos enfocaremos en los siguientes conceptos clave:

  • El uso avanzado de **macros globales** para organizar directorios.
  • La importancia de limpiar variables de texto con `tostring` y `replace`.
  • El uso de la función `cond()` para aplicar lógica condicional en la creación de variables.
  • Comandos cruciales para combinar datos como **`tempfile`** y **`merge`**.
  • Un repaso rápido y esencial de **`preserve`** y **`restore`**.

¡Prepárate para construir tu propia base de conciliación!

1. Macros Globales: Organización de Directorios

Una práctica fundamental para scripts escalables y portátiles.

En la Clase 5, hablamos de las macros locales, que son temporales. Hoy, usaremos **macros globales**, que son aún más poderosas para la organización de proyectos. Una macro global existe a lo largo de toda la sesión de Stata y se llama con un signo de dólar (`$`).

  • `global basedir`: Define la macro global `basedir` con la ruta de tu carpeta principal del curso.
  • `global outputdir "$basedir/output"`: Define una macro para el directorio de salida. Fíjate cómo usa la macro `basedir` para construir la ruta. Esto es lo que hace que tu código sea flexible: si cambias la ruta de la carpeta base, todas las demás rutas se actualizan automáticamente.

******** 1. DEFINICIÓN DE DIRECTORIOS ********
global basedir    "C:\Users\Research assistant\Documents\stata_curso"
global outputdir  "$basedir/output"
global tempdir    "$basedir/temp"

2. Lectura y Limpieza de Datos

Importando desde CSV y preparando la clave de unión.

Importamos nuestro archivo principal de conciliación y realizamos una limpieza esencial de la variable `id_empresa`, que será nuestra clave para unir tablas.

Importar Archivo CSV (`import delimited`)

Usamos `import delimited` para archivos de texto como CSV. La opción `encoding(utf8)` es crucial para evitar problemas con caracteres especiales (ej. acentos, la ñ).


******** 2. LECTURA DE BASE DE CONCILIACIÓN ********
import delimited "$basedir/conciliacion_tributaria_simulada.csv", clear ///
    case(preserve) encoding(utf8)

Limpieza de Variables de Texto (`tostring` y `replace`)

La variable `id_empresa` debe ser un `string` limpio sin espacios para que el `merge` funcione correctamente.

  • `tostring id_empresa, replace force`: Este comando convierte la variable `id_empresa` a tipo `string`. La opción `force` es útil si contiene números que parecen strings.
  • `replace id_empresa = strtrim(stritrim(id_empresa))`: Este comando es clave para la limpieza. `stritrim()` quita los espacios al inicio y al final del texto. Luego, `strtrim()` quita los espacios internos extra. Esto asegura que la clave `id_empresa` sea idéntica en ambas tablas.


* Asegurar que id_empresa esté limpia y sea string
tostring id_empresa, replace force
replace id_empresa = strtrim(stritrim(id_empresa))

3. Evaluación de Missings y Ceros

Una revisión fundamental antes de la construcción de variables.

Antes de construir nuestras variables, es vital entender el estado de nuestros datos. Este bucle `foreach` nos ayuda a contar los valores perdidos (`missing`) y los ceros para cada variable numérica, excluyendo las de identificación y año.


******** 3. EVALUACIÓN DE MISSINGS Y CEROS ********
ds año id_empresa actividad_economica, not
foreach var of varlist `r(varlist)' {
    count if missing(`var')
    di "`var': missing = `r(N)'"

    count if `var' == 0
    di "`var': ceros = `r(N)'"
}

4. Destring y Resumen de Variables

Asegurando que todas las variables numéricas estén en el formato correcto.

Este paso es similar al que vimos en la Clase 5, donde usamos un bucle `foreach` para convertir todas las variables que no son de identificación o año a un formato numérico. Luego, usamos `summarize` para obtener un resumen rápido de todas las variables numéricas.


******** 4. DESTRING DE VARIABLES NUMÉRICAS ********
ds año id_empresa actividad_economica, not
foreach var of varlist `r(varlist)' {
    capture destring `var', replace ignore(",") force
}

******** 5. RESUMEN DE VARIABLES NUMÉRICAS ********
ds año actividad_economica, not
summarize `r(varlist)'

5. Construcción de Variables de Conciliación

Usando la función `cond()` para una lógica avanzada.

Aquí es donde construimos las variables que necesitamos para la conciliación. En lugar de `if` y `else` por separado, usamos la función **`cond()`**, que es ideal para aplicar lógica condicional en la creación de variables en una sola línea.

  • La sintaxis es: **`cond(condición, valor_si_verdadero, valor_si_falso)`**.
  • En este caso, la condición es `missing(variable)`, que evalúa si la variable tiene un valor perdido. Si es así, asigna `0` (`valor_si_verdadero`), y si no lo es, asigna el valor original de la variable (`valor_si_falso`). Esto nos permite tratar los missings como ceros para los cálculos.


******** 6. CONSTRUCCIÓN DE VARIABLES DE CONCILIACIÓN ********

gen utilidad_contable = utilidad_antes_participacion

* Diferencias permanentes
replace participacion_trabajadores    = cond(missing(participacion_trabajadores), 0, participacion_trabajadores)
replace gastos_no_deducibles        = cond(missing(gastos_no_deducibles), 0, gastos_no_deducibles)
replace ingresos_exentos              = cond(missing(ingresos_exentos), 0, ingresos_exentos)
replace ajuste_rimpe                  = cond(missing(ajuste_rimpe), 0, ajuste_rimpe)

gen dif_perm_total = participacion_trabajadores + gastos_no_deducibles - ingresos_exentos - ajuste_rimpe

* Diferencias temporales
gen prov_inventarios    = cond(missing(provision_inventarios), 0, provision_inventarios)
gen amortizaciones      = cond(missing(amortizaciones_acumuladas), 0, amortizaciones_acumuladas)

gen dif_temp_total = prov_inventarios + amortizaciones

6. Cálculo de la Utilidad Gravable

La fórmula final de la conciliación.

El objetivo final es calcular la utilidad gravable, sumando las diferencias permanentes y temporales a la utilidad contable.


******** 7. CÁLCULO DE UTILIDAD GRAVABLE ********
gen utilidad_gravable = utilidad_contable + dif_perm_total + dif_temp_total if !missing(utilidad_contable)

7. Merge (Join): Uniendo Tablas

Combinando datos de la base principal con información del RUC.

Aquí aprenderás un concepto avanzado y muy útil: unir dos bases de datos. Para hacer esto de manera segura, usaremos **`tempfile`** y **`merge`**.

  • `tempfile nombre`: Este comando crea un nombre de archivo temporal para la base de datos que está actualmente en memoria. Esto es fundamental para no sobrescribir archivos importantes y para asegurarnos de que la base de datos principal esté disponible más tarde. En el código, guardamos la base principal con `save `tempfile'', la base RUC con `save `ructable'' y luego volvemos a la base principal con `use `masterdata'`.
  • `merge m:1`: Este es el comando para unir bases de datos.
    • `m:1`: Indica una relación "muchos a uno", donde `id_empresa` puede aparecer muchas veces en la base principal (`master`) pero solo una vez en la base RUC (`using`).
    • `id_empresa`: Es la clave de unión, la variable que debe ser idéntica en ambas bases para que los datos se puedan combinar.
    • `using `nombre_base'`: Indica la base de datos con la que queremos hacer el `merge`.
    • `keep(match master)`: Le dice a Stata que mantenga solo las observaciones que encontraron una coincidencia (`match`) o que existían solo en la base principal (`master`).
    • `nogenerate`: Es una opción que evita que Stata cree la variable `_merge`, que indica el resultado de la unión.


******** 8. JOIN CON TABLA RUC (TIPO CONTRIBUYENTE, GC) ********

* Guardar base principal antes del merge
tempfile masterdata
save `masterdata', replace

* Cargar y preparar tabla RUC
import delimited "$basedir/tabla_ruc_simulada.csv", clear encoding(utf8)
tostring id_empresa, replace force
replace id_empresa = strtrim(stritrim(id_empresa))

keep id_empresa tipo_contribuyente es_gran_contribuyente
tempfile ructable
save `ructable', replace

* Volver a cargar la base principal
use `masterdata', clear
tostring id_empresa, replace force
replace id_empresa = strtrim(stritrim(id_empresa))

* Merge con tabla RUC
merge m:1 id_empresa using `ructable', keep(match master) nogenerate

8. Resúmenes de Utilidad Gravable

Calculando estadísticas por grupo y guardando la base final.

Finalmente, resumimos la utilidad gravable por categorías, lo que nos permite comparar los resultados de la conciliación entre diferentes tipos de contribuyentes y años.

Resumen por Grupo (`collapse` con `preserve`/`restore`)

El comando **`collapse`** es la forma de Stata de crear una base de datos resumen, con una observación por cada grupo definido.

  • `collapse (mean) media = utilidad_gravable`: Calcula la media de `utilidad_gravable` y la guarda en una nueva variable llamada `media`.
  • `by(año tipo_contribuyente ...)`: Define las variables de agrupación. La base de datos resultante tendrá una fila para cada combinación única de estas variables.
  • `preserve` y `restore`: Recordamos de la Clase 5 que este par de comandos es esencial. Como `collapse` modifica la base de datos de manera irreversible, **`preserve`** guarda la base original antes del `collapse` y **`restore`** la recupera después de que hayamos terminado, permitiéndonos guardar la base de datos completa con todos los cálculos.


******** 9. RESÚMENES DE UTILIDAD GRAVABLE ********
preserve
collapse (mean) media = utilidad_gravable ///
         (sd) sd = utilidad_gravable ///
         (count) conteo = utilidad_gravable, ///
         by(año tipo_contribuyente es_gran_contribuyente)

list año tipo_contribuyente es_gran_contribuyente ///
      media sd conteo, sepby(año)
restore

9. Guardar la Base Final

Concluyendo el proceso ETL.

Finalmente, guardamos la base de datos completa y construida. Fíjate cómo usamos la macro global `$outputdir` que definimos al inicio. Esto asegura que nuestro archivo de salida se guarde en la ubicación correcta, sin importar en qué computadora se ejecute el script, siempre y cuando la macro base esté bien definida.


******** 10. GUARDAR BASE FINAL ********
save "$outputdir/conciliacion_construida.dta", replace
